# 🔧 Система управления автосервисом — Версия 1

**Версия:** 1.0 (auth-only)  
**Статус:** Только авторизация и регистрация

Веб-приложение для автоматизации работы сети автомастерских. Данная версия включает **только авторизацию и регистрацию клиентов**.

---

## 🚀 Быстрый старт

### Запуск одной командой:

```bash
docker-compose up --build
```

**ВСЁ!** Система автоматически:
- ✅ Создаст базу данных
- ✅ Применит миграции
- ✅ Создаст 1 администратора
- ✅ Создаст 7 мастерских в разных городах

### 🌐 Доступ к приложению:

| Сервис | URL |
|--------|-----|
| **Frontend** | http://localhost:3000 |
| **Backend API** | http://localhost:8000 |
| **Swagger Docs** | http://localhost:8000/docs |

---

## 🔐 Учётные данные

### Администратор (глобальный):
| Email | Пароль | Права |
|-------|--------|-------|
| `admin@autoservice.ru` | `admin123` | Полный доступ ко всем функциям системы |

### Клиенты:
Клиенты могут **самостоятельно зарегистрироваться** на странице `/register`.

Для регистрации требуется:
- Имя и фамилия
- Email
- Телефон (опционально)
- Выбор автосервиса (город)
- Пароль (минимум 6 символов)

## ⚠️ Важно!

- При первом запуске система **автоматически** создаст все начальные данные
- При повторном запуске данные **не будут созданы заново** (проверка по наличию пользователей)
- Для полного сброса используйте `docker-compose down -v`
- Техники привязаны к мастерским — нельзя назначить техника из другой мастерской

## 🐛 Решение проблем

### Ошибка «port already in use»:
```bash
# Проверьте что порты свободны
netstat -ano | findstr :3000
netstat -ano | findstr :8000
netstat -ano | findstr :5432

# Остановите процессы или измените порты в docker-compose.yml
```

### Ошибка подключения к БД:
```bash
# Полный сброс
docker-compose down -v
docker-compose up --build
```

### Логи для отладки:
```bash
# Все логи
docker-compose logs -f

# Логи конкретного сервиса
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f db
```
