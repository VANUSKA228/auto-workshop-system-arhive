## 1. Общие сведения

### 1.1 Назначение системы
Веб-приложение предназначено для автоматизации работы сети автомастерских: управления заявками клиентов, контроля работы мастеров смены и администрирования системы. Все три роли работают в едином интерфейсе.

### 1.2 Технологический стек

| Слой | Технология | Версия / Пакеты |
|------|------------|-----------------|
| Backend | Python + FastAPI | Python 3.11+, FastAPI 0.134+, Pydantic v2, SQLAlchemy 2.0, Alembic |
| База данных | PostgreSQL | PostgreSQL 15+, psycopg2-binary |
| Аутентификация | JWT | python-jose, bcrypt |
| Frontend | React + TypeScript | React 18, TypeScript 5, React Router v6, Axios, React Hook Form |
| Стилизация | Tailwind CSS | Tailwind CSS 3 |
| Сборка | Vite | Vite 5 |
| Деплой | Docker | Docker + Docker Compose |

### 1.3 Бизнес-роли (3 роли)

| Роль | Описание | Основные права |
|------|----------|----------------|
| Клиент | Владелец автомобиля, обратившийся в сервис | Создание заявки, просмотр своих заявок, «оплата» |
| Мастер смены | Сотрудник, принимающий и выполняющий заявки | Просмотр заявок своей мастерской, обработка, смена статуса |
| Администратор | Управляющий сервисом | Полный доступ: пользователи, заявки, отчёты, услуги, мастерские |

## 3. База данных (PostgreSQL)

### 3.1 Основные таблицы
- `roles` — роли (client, master, admin)
- `users` — пользователи (с полем workshop_id для привязки к филиалу)
- `workshops` — мастерские (Москва, Санкт-Петербург, Новосибирск)
- `services` — классификатор услуг
- `orders` — заявки (client_id, master_id, workshop_id)
- `order_services` — связь заявка–услуга
- `payments` — заглушки платежей
- `workers` — работники (механики)

### 3.2 Логика привязки к мастерской
- **User.workshop_id** — мастерская, к которой привязан пользователь (клиент или мастер)
- **Order.workshop_id** — мастерская, в которой обслуживается заявка
- При создании заявки: `order.workshop_id = user.workshop_id` (заявка идёт в мастерскую клиента)
- Мастер видит только заявки своей мастерской: `Order.workshop_id == user.workshop_id`

## 4. Backend — REST API

### 4.1 Авторизация
- Заголовок: `Authorization: Bearer <JWT_ACCESS_TOKEN>`
- JWT содержит: user_id, role, exp (8 часов)
- `get_current_user()` — декодирует JWT, возвращает User
- `role_required(*roles)` — проверка роли

### 4.2 Эндпоинты (реализовано)

| Метод | URL | Доступ | Описание |
|-------|-----|--------|----------|
| POST | /auth/login | Все | Вход: email + password → JWT |
| POST | /auth/register | Admin | Создание пользователя (мастер/клиент) |
| **POST** | **/auth/register/client** | **Все** | **Самостоятельная регистрация клиента (с выбором мастерской)** |
| GET | /orders/ | Master, Admin | Список заявок (Master — только своей мастерской) |
| GET | /orders/my | Client | Список заявок текущего клиента |
| POST | /orders/ | All | Создание заявки (workshop_id берётся из user) |
| GET | /orders/{id} | Master, Admin, Client(свои) | Детали заявки |
| PATCH | /orders/{id} | Master, Admin | Редактирование заявки |
| DELETE | /orders/{id} | Master, Admin | Удаление заявки |
| GET | /services/ | All | Список услуг |
| POST | /services/ | Admin | Создание услуги |
| GET | **/workshops/public** | **Все (без авторизации)** | **Публичный список мастерских для формы регистрации** |
| GET | /workshops/ | Admin | Список мастерских |
| GET | /users/ | Admin | Список пользователей |
| POST | /payments/stub | Client | Заглушка оплаты |

### 4.3 Регистрация клиента (POST /auth/register/client)

**Тело запроса:**
```json
{
  "first_name": "Иван",
  "last_name": "Петров",
  "email": "client@mail.ru",
  "phone": "+7 999 123-45-67",
  "password": "secret123",
  "workshop_id": 1
}
```

**Логика:**
- Клиент при регистрации **обязательно выбирает мастерскую** (город/филиал) из списка
- `workshop_id` — ID мастерской (Москва, Санкт-Петербург, Новосибирск)
- Все заявки клиента автоматически попадают в выбранную мастерскую
- Мастер этой мастерской видит и обрабатывает заявки

**Ответ:** `{ token, user }` — как при логине, клиент сразу авторизован

## 5. Frontend — реализовано

### 5.1 Роутинг
- `/` — LandingPage (главная, кнопки Войти / Регистрация)
- `/login` — LoginPage
- `/register` — ClientRegisterPage (форма регистрации с выбором автосервиса)
- `/app/orders` — OrdersPage
- `/app/orders/new` — CreateOrderPage
- `/app/payment` — PaymentPage (заглушка)
- `/app/services` — ServicesPage
- `/app/users` — UsersPage (только Admin)
- ProtectedRoute, RoleRoute — защита маршрутов

### 5.2 Страница регистрации клиента (/register)

**Реализовано:**
- Форма: ФИО, email, телефон, **выбор автосервиса (город)**, пароль, подтверждение пароля
- Список мастерских загружается из GET /workshops/public при открытии страницы
- Валидация: React Hook Form + Yup (workshop_id — обязательное поле)
- После успешной регистрации — автологин и редирект на /app/orders
